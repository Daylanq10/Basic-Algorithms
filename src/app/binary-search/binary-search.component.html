Binary Search

This search algorithm is a more efficient search that begins in the middle of the list and continually divides in the direction of the result until a result is found. This search algorithm requires a sorted list.

Process

1. Start with the middle index.

2. If the item is not found, move to the index halfway from the current middle index toward the side of the list that is closer to the desired item.

3. Continuouslly repeat the second step until a result is found, or until the list can no longer be divided and the item is not found.

4. If the item is found return the index and stop searching.

5. If the item is not in the list, return -1.

Complexities

Time Complexity - O(Log(n))

Space Complexity - Increases as list size increases.
